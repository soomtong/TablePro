# Kết nối Redis

TablePro cung cấp hỗ trợ cho Redis 6.0 và các phiên bản mới hơn. Redis là kho dữ liệu key-value trong bộ nhớ, được sử dụng làm cơ sở dữ liệu, cache và message broker. Các key được nhóm theo namespace phân tách bởi dấu hai chấm trong sidebar, và các giá trị được hiển thị với định dạng phù hợp theo kiểu dữ liệu trong data grid.

## Thiết lập Nhanh

<Steps>
  <Step title="Mở Biểu mẫu Kết nối">
    Nhấp **New Connection** từ màn hình Chào mừng hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn Redis">
    Chọn **Redis** từ trình chọn loại cơ sở dữ liệu
  </Step>
  <Step title="Nhập Thông tin Kết nối">
    Điền host, port, password (nếu cần) và database index
  </Step>
  <Step title="Kiểm tra và Kết nối">
    Nhấp **Test Connection**, sau đó **Create**
  </Step>
</Steps>

## Cài đặt Kết nối

### Các trường Bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Định danh kết nối | - |
| **Host** | Hostname hoặc IP của máy chủ | `localhost` |
| **Port** | Cổng máy chủ Redis | `6379` |

### Các trường Tùy chọn

| Trường | Mô tả |
|-------|-------------|
| **Password** | Mật khẩu máy chủ Redis (AUTH) |
| **Database** | Chỉ số database (0-15, mặc định `0`) |

<Tip>
Đối với phát triển local không bật xác thực, bạn có thể để trống trường password. Redis sử dụng database index `0` theo mặc định.
</Tip>

## Định dạng URL Kết nối

Bạn có thể import kết nối bằng Redis URL:

| Định dạng | Ví dụ |
|--------|---------|
| Redis | `redis://password@host:6379/0` |
| Redis (không xác thực) | `redis://host:6379/0` |
| Redis TLS | `rediss://password@host:6380/0` |

Scheme `rediss://` (lưu ý hai chữ `s`) chỉ định kết nối được mã hóa TLS.

## Ví dụ Cấu hình

### Máy chủ Phát triển Local

```
Name:     Local Redis
Host:     localhost
Port:     6379
Password: (để trống)
Database: 0
```

### Docker Redis Container

```
Name:     Docker Redis
Host:     localhost
Port:     6379 (hoặc cổng được ánh xạ của bạn)
Password: (mật khẩu từ biến môi trường REDIS_PASSWORD)
Database: 0
```

### Redis Cloud / Dịch vụ Quản lý

```
Name:     Redis Cloud
Host:     redis-12345.c1.us-east-1-2.ec2.cloud.redislabs.com
Port:     12345
Password: (mật khẩu Redis Cloud của bạn)
Database: 0
```

<Note>
Đối với các dịch vụ Redis quản lý, SSL/TLS thường là bắt buộc. Bật nó trong phần SSL/TLS của biểu mẫu kết nối.
</Note>

### Máy chủ Từ xa

```
Name:     Production Cache
Host:     redis.example.com
Port:     6379
Password: (mật khẩu bảo mật)
Database: 0
```

<Warning>
Đối với các instance Redis production, hãy xem xét sử dụng [SSH tunneling](/vi/databases/ssh-tunneling) để kết nối an toàn.
</Warning>

## Kết nối SSL/TLS

TablePro hỗ trợ mã hóa SSL/TLS cho kết nối Redis. Cấu hình SSL trong phần **SSL/TLS** của biểu mẫu kết nối.

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL |
| **Required** | Bắt buộc mã hóa SSL |
| **Verify CA** | Bắt buộc SSL và xác minh chứng chỉ server với CA |

Đối với chế độ **Verify CA**, cung cấp đường dẫn đến file chứng chỉ CA. Bạn cũng có thể cung cấp file chứng chỉ client và khóa tùy chọn để xác thực TLS hai chiều.

<Tip>
Sử dụng scheme URL `rediss://` khi import kết nối yêu cầu TLS.
</Tip>

<Note>
Nếu bạn không muốn cấu hình chứng chỉ SSL, [SSH tunneling](/vi/databases/ssh-tunneling) là phương án thay thế mã hóa toàn bộ traffic qua đường hầm SSH.
</Note>

## Hỗ trợ SSH Tunnel

Bạn có thể kết nối đến Redis qua SSH tunnel để truy cập an toàn đến máy chủ từ xa. Xem [SSH Tunneling](/vi/databases/ssh-tunneling) để biết hướng dẫn thiết lập chi tiết.

## Tính năng

### Duyệt Namespace

Sau khi kết nối, sidebar nhóm các key theo tiền tố phân tách bởi dấu hai chấm. Ví dụ, các key có tên `user:1`, `user:2` và `session:abc` xuất hiện dưới namespace `user` và `session`:

1. Nhấp vào tên kết nối trong sidebar
2. Mở rộng namespace để xem các key
3. Nhấp vào key để xem giá trị

Các key không có dấu hai chấm phân tách được nhóm dưới namespace gốc.

### Xem Key-Value

Các giá trị được hiển thị trong data grid với định dạng phù hợp theo kiểu:

- Giá trị **String** được hiển thị dạng văn bản thuần
- Các trường **Hash** được hiển thị dạng cặp key-value theo hàng
- Các phần tử **List** được hiển thị với chỉ số
- Các thành viên **Set** được hiển thị dạng hàng riêng lẻ
- Các thành viên **Sorted Set** được hiển thị cùng với điểm số
- Giá trị **JSON** được tô sáng cú pháp

### Quản lý TTL

Xem và quản lý thời gian sống (TTL) cho các key:

- TTL được hiển thị bên cạnh mỗi key trong data grid
- Key không có hạn sử dụng hiển thị `-1`
- Key không tồn tại hiển thị `-2`
- Đặt hoặc cập nhật giá trị TTL trực tiếp từ giao diện

### Redis CLI

Thực thi lệnh Redis trực tiếp trong tab editor sử dụng cú pháp lệnh Redis chuẩn:

```redis
-- Thao tác key
GET mykey
SET mykey "hello" EX 60
DEL mykey key2 key3
KEYS user:*

-- Thao tác hash
HGETALL myhash
HSET myhash field1 "value1"
HDEL myhash field1

-- Thao tác list
LRANGE mylist 0 -1
LPUSH mylist "item1" "item2"
LLEN mylist

-- Thao tác set
SMEMBERS myset
SADD myset "member1" "member2"
SCARD myset

-- Thao tác sorted set
ZRANGE myzset 0 -1 WITHSCORES
ZADD myzset 1 "one" 2 "two"
ZCARD myzset

-- Quét key
SCAN 0 MATCH user:* COUNT 100

-- Thông tin server
PING
INFO
DBSIZE
```

### Các Lệnh Được Hỗ trợ

TablePro hỗ trợ toàn bộ tập lệnh Redis, bao gồm:

| Danh mục | Lệnh |
|----------|----------|
| **String** | GET, SET, MGET, MSET, INCR, DECR, APPEND |
| **Hash** | HGET, HSET, HGETALL, HDEL, HKEYS, HVALS, HLEN |
| **List** | LPUSH, RPUSH, LPOP, RPOP, LRANGE, LLEN, LINDEX |
| **Set** | SADD, SREM, SMEMBERS, SISMEMBER, SCARD, SUNION |
| **Sorted Set** | ZADD, ZREM, ZRANGE, ZRANGEBYSCORE, ZCARD, ZSCORE |
| **Key** | DEL, EXISTS, EXPIRE, TTL, TYPE, RENAME, KEYS, SCAN |
| **Server** | PING, INFO, DBSIZE, FLUSHDB, SELECT |

Các lệnh không được liệt kê ở trên vẫn được gửi đến server và phản hồi được hiển thị trong bảng kết quả.

## Khắc phục sự cố

### Kết nối Bị từ chối

**Triệu chứng**: "Connection refused" hoặc timeout

**Nguyên nhân và Giải pháp**:

1. **Máy chủ Redis không chạy**
   ```bash
   # Kiểm tra xem Redis có đang chạy không
   redis-cli ping

   # Khởi động Redis (macOS với Homebrew)
   brew services start redis
   ```

2. **Cổng sai**
   - Xác minh cổng trong file cấu hình Redis (`redis.conf`)
   - Kiểm tra cấu hình của bạn

3. **Redis không lắng nghe trên địa chỉ mong đợi**
   - Kiểm tra chỉ thị `bind` trong `redis.conf`
   - Đối với kết nối từ xa, thêm IP client hoặc sử dụng `0.0.0.0`

### Xác thực Thất bại

**Triệu chứng**: "NOAUTH Authentication required" hoặc "ERR invalid password"

**Giải pháp**:

1. Xác minh mật khẩu khớp với `requirepass` trong `redis.conf`
2. Đối với Redis 6.0+ với ACL:
   ```redis
   ACL LIST
   ACL SETUSER myuser on >password ~* +@all
   ```

### Hết thời gian Kết nối

**Triệu chứng**: Kết nối bị treo hoặc hết thời gian

**Giải pháp**:

1. Xác minh host và port chính xác
2. Kiểm tra kết nối mạng và quy tắc firewall
3. Đối với Redis được host trên cloud, đảm bảo IP của bạn được thêm vào danh sách trắng

## Hạn chế Đã biết

- **Cluster mode**: Redis Cluster với sharding tự động không được hỗ trợ. Kết nối trực tiếp đến từng node riêng lẻ.
- **Pub/Sub**: Đăng ký Pub/Sub thời gian thực không được hỗ trợ trong phiên bản hiện tại.
- **Streams**: Redis Streams (XADD, XREAD, v.v.) không được hỗ trợ trong data grid, nhưng có thể thực thi lệnh trong CLI.
- **Modules**: Các lệnh RedisJSON, RediSearch và module khác có thể được thực thi qua CLI nhưng có thể không có hỗ trợ UI chuyên biệt.
- **Key lớn**: Key có hàng triệu phần tử (list, set hoặc hash lớn) được phân trang để tránh vấn đề bộ nhớ.

## Mẹo Hiệu suất

### Database Lớn

Đối với các instance Redis có hàng triệu key:

1. Sử dụng duyệt namespace để lọc key theo tiền tố
2. Sử dụng `SCAN` thay vì `KEYS` để tìm kiếm theo mẫu trong CLI
3. Phân trang được áp dụng tự động khi duyệt collection lớn

### Giám sát Bộ nhớ

Kiểm tra mức sử dụng bộ nhớ cho instance Redis:

```redis
INFO memory
MEMORY USAGE mykey
DBSIZE
```
