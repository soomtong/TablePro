# Nhập & Xuất

Export dữ liệu theo năm định dạng (CSV, JSON, SQL, MQL, XLSX), import file SQL có hỗ trợ gzip, và dán dữ liệu dạng bảng từ clipboard vào grid.

## Export Dữ Liệu

### Tùy Chọn Export

Export dữ liệu từ kết quả query hoặc bảng:

1. Chạy query hoặc mở bảng
2. Click nút **Export** trong toolbar (hoặc nhấn `Cmd+Shift+E`)
3. Chọn định dạng export
4. Chọn bảng cần export trong tree view
5. Cấu hình tùy chọn cho từng định dạng
6. Click **Export**

<Note>
**MongoDB**: Export SQL không khả dụng cho MongoDB. Dùng CSV, JSON, MQL hoặc XLSX thay thế. MQL tạo script `db.collection.insertMany([...])` chạy trực tiếp trong `mongosh`.
</Note>

<Note>
**Redis**: Export SQL và MQL không khả dụng cho Redis. Dùng CSV, JSON hoặc XLSX để export dữ liệu key-value.
</Note>

### Export Kết Quả Query

Để export kết quả query ra file:

1. Viết và chạy query trong trình soạn thảo SQL
2. Chuột phải vào bảng kết quả và chọn **Export Results...**, hoặc vào **File** > **Export Results...**
3. Chọn định dạng export (CSV, JSON, SQL, XLSX hoặc MQL)
4. Cấu hình tùy chọn và tên file
5. Click **Export**

Hộp thoại export hiển thị số hàng và dùng giao diện đơn giản không có tree chọn bảng, vì chỉ export kết quả query trong bộ nhớ.

<Tip>
Với export lớn, dùng phân trang hoặc LIMIT trong query để kiểm soát kích thước output.
</Tip>

### Export Toàn Bộ Bảng

Để export tất cả dữ liệu từ bảng:

1. Click bảng trong sidebar
2. Click **Export** không áp dụng filter
3. Hoặc chạy `SELECT * FROM table_name` rồi export

<Tip>
Export nhiều bảng cùng lúc bằng cách chọn trong tree view. Khi export SQL, mỗi bảng có tùy chọn riêng cho structure (DDL), DROP statements và data.
</Tip>

### Export Chỉ Cấu Trúc

Để export chỉ cấu trúc bảng (DDL):

1. Mở tab **Structure** của bảng
2. Đi đến tab con **DDL**
3. Sao chép câu lệnh CREATE TABLE

Hoặc dùng lệnh theo database:

```sql
-- MySQL
SHOW CREATE TABLE users;

-- PostgreSQL
pg_dump -s -t users database_name
```

### Tiến Trình Export

Khi export bảng lớn, thanh tiến trình hiển thị:

- **Số hàng đã xử lý** trên tổng hàng
- **Bảng đang export** (khi export nhiều bảng)

## Dán từ Clipboard (CSV/TSV)

Dán dữ liệu dạng bảng từ clipboard vào data grid. TablePro tự động phát hiện CSV (phân tách dấu phẩy) hay TSV (phân tách tab).

**Cách dùng:**

1. Sao chép dữ liệu từ bảng tính, text editor hoặc nguồn khác
2. Chọn một hàng trong data grid
3. Nhấn `Cmd+V` để dán

CSV parser tuân thủ RFC 4180, hỗ trợ:

- Trường có dấu phẩy bên trong (ví dụ: `"New York, NY"`)
- Dấu ngoặc kép thoát (ví dụ: `"He said ""hello"""`)
- Giá trị nhiều dòng trong trường trích dẫn
- Tự động phát hiện hàng tiêu đề (bỏ qua nếu tên cột khớp)

<Tip>
Định dạng tự động phát hiện: nhiều tab hơn dấu phẩy thì parse TSV, ngược lại CSV. Dữ liệu từ Excel hoặc Google Sheets thường là tab-separated.
</Tip>

## Import Dữ Liệu

TablePro import file **SQL** (`.sql` và `.sql.gz`). Câu lệnh chạy trực tiếp trên database, phù hợp để khôi phục backup, chạy migration hoặc seeding dữ liệu.

<Note>
**MongoDB**: Import SQL không khả dụng cho MongoDB. Dùng `mongoimport` hoặc MQL shell.
</Note>

### Quy Trình Import

<Steps>
  <Step title="Mở hộp thoại Import">
    Click **File** > **Import** (hoặc nhấn `Cmd+Shift+I`). Cũng có thể kéo thả file `.sql` hoặc `.sql.gz` vào ứng dụng.
  </Step>
  <Step title="Chọn file SQL">
    Chọn file `.sql` hoặc `.sql.gz` từ máy tính.
  </Step>
  <Step title="Cấu hình tùy chọn">
    Đặt encoding, transaction wrapping và tùy chọn foreign key check.
  </Step>
  <Step title="Xem trước SQL">
    Kiểm tra SQL preview, số câu lệnh và kích thước file trước khi import.
  </Step>
  <Step title="Import">
    Click **Import** để chạy các câu lệnh và theo dõi tiến trình.
  </Step>
</Steps>

### Tùy Chọn Import

| Tùy chọn | Mô tả | Mặc định |
|--------|-------------|---------|
| Encoding | Encoding file: UTF-8, UTF-16, Latin1, ASCII | UTF-8 |
| Wrap in transaction | Chạy tất cả câu lệnh trong một transaction | Có |
| Disable foreign key checks | Tạm tắt kiểm tra FK trong quá trình import | Có |

<Tip>
Bọc trong transaction đảm bảo nếu câu lệnh nào lỗi, tất cả thay đổi rollback, giữ database nhất quán.
</Tip>

<Tip>
Các tùy chọn import (gói trong transaction, tắt kiểm tra foreign key) được lưu lại giữa các lần khởi động ứng dụng. Chỉ cần thay đổi một lần.
</Tip>

### Xem Trước SQL

Trước khi import, hộp thoại hiển thị:

- **SQL code preview** nội dung file
- **Số câu lệnh** -- tổng số câu lệnh SQL phát hiện
- **Kích thước file**

Giúp kiểm tra file chứa đúng SQL mong đợi trước khi chạy.

## Tiến Trình và Lỗi

### Tiến Trình Import

Trong quá trình import, thanh tiến trình hiển thị:

- **Câu lệnh đã xử lý** trên tổng câu lệnh
- **Câu lệnh đang chạy** (rút gọn 50 ký tự)
- **Thanh tiến trình** phần trăm hoàn thành

### Xử Lý Lỗi

Import dừng ngay khi gặp lỗi đầu tiên. Nếu bật transaction, tất cả thay đổi tự động rollback.

Khi lỗi xảy ra, TablePro hiển thị:

- **Câu lệnh lỗi** -- SQL gây ra lỗi
- **Số dòng** -- vị trí trong file
- **Thông báo lỗi** -- lỗi từ database

<Warning>
Không có chế độ "bỏ qua lỗi". Nếu gặp lỗi, import dừng. Bật "Wrap in transaction" để partial import rollback sạch.
</Warning>

## Nén

### Export Nén

Gzip chỉ khả dụng cho **SQL export**:

- Bật "Compress with gzip" trong tùy chọn SQL export
- Output lưu dạng `.sql.gz`
- Giảm đáng kể kích thước file cho SQL dump lớn

<Note>
Nén gzip không khả dụng cho CSV, JSON hoặc XLSX.
</Note>

### Import Nén

- TablePro tự động phát hiện file `.sql.gz`
- Giải nén tức thì trong quá trình import
- Hoạt động giống import file `.sql` thường

## Thực Hành Tốt Nhất

### Export

1. **Lọc trước**: Chỉ export những gì cần
   ```sql
   SELECT * FROM orders WHERE status = 'completed' AND year(created_at) = 2024;
   ```

2. **Cân nhắc kích thước**: Dùng LIMIT cho bảng rất lớn
   ```sql
   SELECT * FROM logs ORDER BY created_at DESC LIMIT 100000;
   ```

3. **Chọn định dạng phù hợp**:
   - CSV cho phân tích dữ liệu
   - JSON cho API/web
   - SQL cho di chuyển database
   - MQL cho MongoDB backup/restore qua `mongosh`
   - XLSX cho chia sẻ với người không kỹ thuật qua Excel

4. **Dùng gzip cho SQL export lớn**: Nén giảm kích thước file đáng kể

### Import

1. **Sao lưu trước**: Luôn backup database trước khi import SQL

2. **Test với file nhỏ trước**: Xác minh tùy chọn import hoạt động đúng trên một phần trước khi chạy import lớn

3. **Dùng transaction**: Giữ "Wrap in transaction" bật để import lỗi tự động rollback

4. **Tắt foreign key checks**: Khi import dump tham chiếu bảng theo thứ tự, giữ bật để tránh lỗi constraint

5. **Kiểm tra encoding**: Đảm bảo encoding khớp với nội dung thực tế của file SQL để tránh lỗi ký tự

## Khắc Phục Sự Cố

### Vấn Đề Export

**"File quá lớn"**
- Dùng LIMIT trong query
- Export theo batch
- Bật nén gzip cho SQL export
- Chọn định dạng nhỏ gọn hơn (CSV vs JSON)

**"Lỗi encoding ký tự"**
- Kiểm tra encoding database
- Export với UTF-8
- Kiểm tra ký tự đặc biệt

### Vấn Đề Import

**"Syntax error trong SQL"**
- Kiểm tra file SQL đúng cú pháp
- Xác minh file tạo cho đúng loại database (MySQL vs PostgreSQL vs SQLite)
- Kiểm tra encoding file khớp tùy chọn đã chọn

**"Duplicate key error"**
- Dữ liệu trùng với primary key hoặc unique key đã có
- Thêm DROP TABLE statements vào file SQL
- Hoặc xóa bảng đích trước khi import

**"Foreign key constraint"**
- Bật "Disable foreign key checks" trong tùy chọn import
- Hoặc đảm bảo bảng cha tạo trước bảng con trong file SQL

**"Transaction bị rollback"**
- Câu lệnh trong file gây lỗi, rollback toàn bộ import
- Kiểm tra thông báo lỗi và số dòng để tìm câu lệnh lỗi
- Sửa file SQL và import lại
