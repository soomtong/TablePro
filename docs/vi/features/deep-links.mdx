TablePro hỗ trợ hai cách mở kết nối từ ứng dụng bên ngoài: URL cơ sở dữ liệu tiêu chuẩn (như `mysql://...`) và URL scheme tùy chỉnh `tablepro://`.

## URL cơ sở dữ liệu

Mở bất kỳ kết nối cơ sở dữ liệu nào trực tiếp bằng URL tiêu chuẩn. Hoạt động từ terminal, script, Alfred, Raycast hoặc bất kỳ ứng dụng nào có thể mở URL.

### Các scheme được hỗ trợ

| Scheme | Cơ sở dữ liệu |
|--------|----------------|
| `mysql://` | MySQL / MariaDB |
| `postgresql://`, `postgres://` | PostgreSQL |
| `mongodb://`, `mongodb+srv://` | MongoDB |
| `redis://`, `rediss://` | Redis |
| `mssql://`, `sqlserver://` | SQL Server |
| `sqlite://` | SQLite (đường dẫn tệp cục bộ) |

### Định dạng URL

```
scheme://[username[:password]@]host[:port][/database][?param=value]
```

### Ví dụ

```bash
# MySQL
open "mysql://root@localhost/mydb" -a TablePro

# PostgreSQL với cổng và cơ sở dữ liệu
open "postgresql://admin@db.example.com:5432/analytics" -a TablePro

# MongoDB
open "mongodb://localhost:27017/mydb" -a TablePro

# Redis
open "redis://localhost:6379" -a TablePro

# SQLite (tệp cục bộ)
open "sqlite:///Users/me/data/app.db" -a TablePro
```

### Cách hoạt động

1. **Khớp kết nối đã lưu:** TablePro kiểm tra xem có kết nối đã lưu nào khớp với host, port, database và username của URL không. Nếu tìm thấy, nó sử dụng lại kết nối đó (bao gồm mật khẩu đã lưu).
2. **Khớp phiên đang hoạt động:** Nếu kết nối đang mở, TablePro đưa cửa sổ hiện có lên phía trước thay vì tạo bản sao.
3. **Kết nối tạm thời mới:** Nếu không tìm thấy kết nối khớp, TablePro tạo kết nối tạm thời và kết nối ngay lập tức. Mật khẩu trong URL được sử dụng cho lần kết nối đầu tiên nhưng không được lưu.

### Tham số truy vấn

Bạn có thể truyền thêm tùy chọn qua tham số truy vấn:

| Tham số | Mô tả |
|---------|-------|
| `table` | Mở bảng cụ thể sau khi kết nối |
| `filter_column`, `filter_operation`, `filter_value` | Áp dụng bộ lọc trên bảng đã mở |
| `condition` | Áp dụng điều kiện WHERE SQL thô |
| `schema` | Chuyển sang schema cụ thể (PostgreSQL) hoặc database (MySQL) sau khi kết nối |

```bash
# Mở bảng với bộ lọc
open "mysql://root@localhost/mydb?table=users&filter_column=status&filter_operation=equal&filter_value=active" -a TablePro

# Mở với điều kiện SQL thô
open "postgresql://admin@localhost/mydb?table=orders&condition=total%20%3E%20100" -a TablePro

# Chuyển schema khi kết nối
open "postgresql://admin@localhost/mydb?schema=reporting" -a TablePro
```

<Warning>
Mặc dù mật khẩu có thể được bao gồm trong URL (`mysql://user:pass@host/db`), hãy tránh chia sẻ URL chứa mật khẩu. Ưu tiên sử dụng kết nối đã lưu với mật khẩu được lưu trữ trong Keychain.
</Warning>

## Deep Links tùy chỉnh (tablepro://)

TablePro cũng đăng ký URL scheme `tablepro://` để bạn có thể mở kết nối, bảng và truy vấn trực tiếp từ trình duyệt, terminal, tin nhắn Slack hoặc bất kỳ ứng dụng nào hỗ trợ liên kết.

## Các mẫu URL

### Mở kết nối

```
tablepro://connect/{tên}
```

Mở kết nối đã lưu khớp với `{tên}`. Nếu không tìm thấy kết nối, một thông báo lỗi sẽ hiển thị.

```bash
open "tablepro://connect/Production"
```

### Mở bảng

```
tablepro://connect/{tên}/table/{bảng}
tablepro://connect/{tên}/database/{db}/table/{bảng}
```

Kết nối và mở bảng được chỉ định. Dạng thứ hai cho phép bạn chỉ định cơ sở dữ liệu cụ thể trên kết nối.

```bash
open "tablepro://connect/Production/table/users"
open "tablepro://connect/Production/database/analytics/table/events"
```

### Mở truy vấn

```
tablepro://connect/{tên}/query?sql={sql_đã_mã_hóa}
```

Kết nối và mở tab truy vấn mới với SQL được điền sẵn. SQL phải được mã hóa phần trăm (percent-encoded).

```bash
open "tablepro://connect/Production/query?sql=SELECT%20*%20FROM%20users%20LIMIT%2010"
```

### Nhập kết nối

```
tablepro://import?name={n}&host={h}&port={p}&type={t}&username={u}&database={db}
```

Tạo kết nối mới và mở biểu mẫu kết nối để bạn có thể xem lại thông tin và thêm mật khẩu trước khi kết nối.

```bash
open "tablepro://import?name=Staging&host=db.example.com&port=5432&type=postgresql&username=admin&database=mydb"
```

**Tham số bắt buộc:** `name`, `host`, `type`
**Tham số tùy chọn:** `port` (mặc định là cổng tiêu chuẩn của loại cơ sở dữ liệu), `username`, `database`

Các giá trị `type` được hỗ trợ: `MySQL`, `MariaDB`, `PostgreSQL`, `SQLite`, `MongoDB` (không phân biệt hoa thường).

## Chi tiết hoạt động

- **Khớp tên kết nối** không phân biệt hoa thường. `Production`, `production` và `PRODUCTION` đều khớp cùng một kết nối.
- **Khoảng trắng và ký tự đặc biệt** trong tên được mã hóa phần trăm tự động bởi trình duyệt và macOS.
- **Đã kết nối:** nếu kết nối đang hoạt động, TablePro bỏ qua việc kết nối lại và mở tab được yêu cầu trực tiếp.
- **Ứng dụng chưa chạy:** macOS khởi chạy TablePro trước, sau đó chuyển URL.

<Warning>
Mật khẩu không bao giờ được bao gồm trong URL deep link. Đối với kết nối được nhập, người dùng phải nhập mật khẩu thủ công trong biểu mẫu kết nối.
</Warning>

## Sử dụng từ Terminal

Dùng lệnh `open` để kích hoạt deep link từ terminal:

```bash
# Kết nối
open "tablepro://connect/Production"

# Mở bảng cụ thể
open "tablepro://connect/Production/table/users"

# Chạy truy vấn
open "tablepro://connect/Production/query?sql=SELECT%20count(*)%20FROM%20orders"

# Nhập kết nối mới để xem lại
open "tablepro://import?name=Dev&host=localhost&port=3306&type=mysql&username=root&database=app"
```
